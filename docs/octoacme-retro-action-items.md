# OctoAcme — Retrospective Action-Item Tracker

## Purpose
Track action items generated from retrospectives to ensure improvements are owned, followed up, and measured. See [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md) for process guidance.

## How to use
1. Populate a new row for each action item at the end of every retrospective.
2. Review the table at the start of the next retrospective and at weekly PM syncs.
3. Mark items **Done** when complete; archive closed rows at the end of each quarter.
4. Link this tracker from the project board or sprint notes for visibility.

## Field Definitions

| Field | Description |
|-------|-------------|
| **ID** | Unique identifier (e.g., A-01, A-02) |
| **Source** | Sprint number or retrospective date where the action item originated |
| **Action Item** | Clear, specific improvement action |
| **Owner** | Named individual or role responsible for completing the action |
| **Due Date** | Target completion date |
| **Success Criteria** | How you will know the action item is complete and effective |
| **Status** | Open / In Progress / Done / Deferred |
| **Notes** | Context, blockers, or links to related issues |

---

## Review Cadence
- **Weekly PM sync:** review all Open and In Progress items; surface blockers
- **Each retrospective:** check previous items before adding new ones; celebrate completions
- **Quarterly:** archive Done items; reassess Deferred items

## Prioritization Guidance
- Focus on 2–3 action items per sprint to avoid overload
- Prefer actions that are specific, measurable, and achievable within the next sprint
- Link high-impact actions to backlog issues for full visibility

## Owner & Accountability
- **Maintained by:** Project Manager (PM)
- **Reviewed:** Weekly PM sync and every retrospective
- **Roles reference:** See [Roles and Personas](octoacme-roles-and-personas.md)
